# Uplift Narrator

Internal Woodruff Billing tool. Takes a PDF generated by the Uplift Collator web app and produces a structured LAA enhancement narrative, plus a paste-ready prompt for polishing the narrative in a local LLM (LM Studio).

## What it produces

For an input `case.pdf`, the narrator writes a folder `case-narrative/` containing:

- **`narrative.md`** — the structured Markdown narrative. Every relevant CAG/Spec/CPR citation is in place. The solicitor's verbatim explanations are inserted as Markdown blockquotes (so they survive any LLM polish step unchanged). Usable as-is, but reads formulaically.
- **`narrative-prompt.txt`** — a paste-ready prompt for LM Studio with two clearly-marked blocks:
  - `--- SYSTEM PROMPT ---` — paste into LM Studio's "System Message" field. Contains the legal-safety guardrails (preserve every citation, never invent facts, polish prose only).
  - `--- USER MESSAGE ---` — paste as the first user message in a new chat. Contains case context plus the full skeleton from `narrative.md`.
- **`narrative-input.json`** — the recovered form data, in the same shape as the in-app `formData`. Useful for debugging extraction or re-running the skeleton.

The polished output from LM Studio still needs a human review pass before submission to the LAA. There's an optional second prompt at `prompts/verification.md` that asks an LLM to compare skeleton against polished output and flag any dropped citations or facts.

## Quick start

```bash
cd narrate
./_setup.sh                        # one-time: creates conda env 'uplift-narrate'
./_run.sh path/to/case.pdf         # writes case-narrative/ next to the PDF
```

Windows: `_setup.bat` and `_run.bat`.

To override the output location:

```bash
./_run.sh path/to/case.pdf --out-dir ~/Desktop/case-output
```

## How it works

1. **Extract** (`extract.py`). Opens the PDF with `pdfplumber`, slices it into sections by the UPPERCASE headers the web app produces (CASE DETAILS, PANEL MEMBERSHIP, STAGE 1, STAGE 2, PROPOSED UPLIFT), then regexes out each ticked criterion's label, category title, and "Explanation: …" block. Labels are mapped back to `QUESTION_BLOCKS` keys via a label → key lookup.
2. **Skeleton** (`skeleton.py`). For each ticked key, looks up the matching entry in `NARRATIVE_TEMPLATES` (defined in `../content-data.js` — the single source of truth) and substitutes the live values: `{UPLIFT_PERCENT}`, `{ITEM_OF_WORK}`, `{FEE_EARNER_NAME}`, `{PANEL_NAME}`, and `{USER_EXPLANATION}` (the solicitor's verbatim text rendered as a blockquote). Bracketed placeholders inside the templates (e.g. `[SPECIFY ARGUMENT]`) are intentionally preserved — they cue the LLM polish step (or human reviewer) to draw on the explanation that follows.
3. **Prompts** (`prompts.py` + `prompts/`). Reads the static system prompt and user-message template, substitutes case context and the skeleton.

`content-data.js` is the canonical home of the questionnaire and templates. `templates.py` parses it at runtime via `json5`, so any future edit to the web app's templates flows through automatically.

## Requirements

Python 3.10+ and conda. No system packages needed — `pdfplumber` is pure Python.

Dependencies (installed by `_setup.sh` / `_setup.bat`):
- `pdfplumber` — PDF text extraction
- `json5` — parses the JS-style content-data.js cleanly

## Tests

```bash
conda activate uplift-narrate
python -m unittest discover -s narrate/tests -v
```

Twenty tests covering: parsing of `content-data.js`, label uniqueness, template-key parity, end-to-end extraction against the fixture PDF (`tests/fixtures/sample.pdf`), skeleton citation coverage, and prompt assembly.

## Troubleshooting

**"narrate: unknown panel/criterion label"** — the PDF was generated against an older version of `content-data.js` whose labels have since changed. Either regenerate the PDF from the current web app, or restore the older `content-data.js` long enough to extract.

**Section-header detection misses something** — only happens if the UPPERCASE titles in `script.js:generatePdfSummary` change. Update `SECTION_PATTERNS` in `extract.py` to match.

**LLM keeps dropping citations** — run the verification prompt (`prompts/verification.md`) as a second LM Studio chat. If a particular model consistently drops citations, switch to a more instruction-following model (Qwen/Mistral/Gemma 2 in the 27–32B range work well).
