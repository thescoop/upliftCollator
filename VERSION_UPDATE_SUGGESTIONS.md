# Adding Version Numbers - Implementation Guide

## Recommendations for Version Tracking

### 1. **Add Version Number to JavaScript (RECOMMENDED)**

In `content-data.js`, add at the top:
```javascript
// Version Information
const APP_VERSION = "1.8";
const APP_RELEASE_DATE = "31 May 2025";
const APP_NAME = "Uplift Collator";
```

### 2. **Display Version in the UI**

#### Option A: In the Footer
Add to `index.html` in the footer section:
```html
<div class="wb-footer-inner-constrained">
    <p>&copy; <span id="currentYearFooter"></span> Woodruff Billing Ltd | Version <span id="appVersion">1.8</span></p>
    <!-- existing footer content -->
</div>
```

#### Option B: In the Sidebar (More Subtle)
Add to the sidebar after the LAA guidance version:
```html
<p class="version-info-sidebar">
    <span id="laa-guide-version-info-sidebar"></span><br>
    <small>Tool Version: <span id="appVersionSidebar">1.8</span></small>
</p>
```

### 3. **Update script.js to Display Version**

Add to the initialization section in `script.js`:
```javascript
// Display version information
if (typeof APP_VERSION !== 'undefined') {
    const appVersionEl = document.getElementById('appVersion');
    const appVersionSidebarEl = document.getElementById('appVersionSidebar');
    if (appVersionEl) appVersionEl.textContent = APP_VERSION;
    if (appVersionSidebarEl) appVersionSidebarEl.textContent = APP_VERSION;
}
```

### 4. **Add Version to PDF Output**

In the PDF generation section, modify the footer to include:
```javascript
addTextToPdf(`Generated by: Woodruff Billing Ltd Uplift Tool v${APP_VERSION} (${LAA_GUIDE_VERSION_INFO_CONST})`, {size:8, italic:true, color: defaultFontColor});
```

### 5. **GitHub Release Tags**

When uploading to GitHub:

1. **Create a git tag:**
   ```bash
   git tag -a v1.8 -m "Version 1.8 - Added inline T&C and Help links"
   git push origin v1.8
   ```

2. **Update README.md** with the version number

3. **Create a GitHub Release:**
   - Go to your repository
   - Click "Releases" → "Create a new release"
   - Choose tag: v1.8
   - Release title: "Version 1.8 - Improved Accessibility"
   - Describe changes in the release notes

### 6. **Version Checking (Optional Advanced Feature)**

Add a version check against GitHub:
```javascript
// Optional: Check for updates (only runs in browser, not during PDF generation)
async function checkForUpdates() {
    try {
        const response = await fetch('https://api.github.com/repos/thescoop/upliftCollator/releases/latest');
        const data = await response.json();
        const latestVersion = data.tag_name.replace('v', '');
        
        if (latestVersion > APP_VERSION) {
            console.log(`New version ${latestVersion} available on GitHub`);
            // Could display a subtle notification to the user
        }
    } catch (error) {
        // Silently fail - not critical
    }
}
```

## Benefits of Version Tracking

1. **User Awareness** - Users know which version they're using
2. **Support** - Easier to troubleshoot issues
3. **Update Management** - Clear when updates are needed
4. **Professional** - Shows active maintenance
5. **Audit Trail** - Important for legal/compliance tools
